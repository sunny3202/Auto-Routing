# 개발 로드맵 (Phase 1~5)

---

## 1. Phase 개요

| Phase | 내용 | 예상 공수 | 선행 조건 |
| ----- | ---- | --------- | --------- |
| Phase 1 | routing_core 코어 엔진 구현 | 2~4주 | 없음 |
| Phase 2 | arithmetic-server 구현 | 1~2주 | Phase 1 |
| Phase 3 | 클라이언트 실제 연결 | 1주 | Phase 2 |
| Phase 4 | 통합 검증 + Hookup 검증 레이어 | 2~3주 | Phase 3 |
| Phase 5 | Revit Addin 정합성 확인 | 1주 | Phase 4 + Revit 환경 |

**총 추정:** 7~11주 (MVP는 1~2주 안에 판단 가능)

---

## 2. MVP 개발 순서 (빠른 Go/No-Go 판단)

```
1. VoxelGrid 구현 — numpy 3D boolean array, mark_obstacle, is_blocked (1~2일)
2. PathFinder 기본 A* — 축방향 6방향, State=(vx,vy,vz,dir_idx,turn_count,straight_count), g-score+bending_weight (2~3일)
3. 1단계 진공 RuleSet 연결 — preprocessing.py 출력 → RoutingPipeline 입력 연결 (1일)
4. vaccum_test_1 단일 엔티티 탐색 — 탐색 시간 측정, path 출력 확인 (0.5일)
5. result.json 출력 → VImportApp 파싱 단위 테스트 (0.5일)
→ 총 5~7일이면 Go/No-Go 기능 기준(1~4) 판단 가능
→ 성공률(기준 5) 측정은 42개 전체 실행 후: 추가 1~2일
```

---

## 3. Go/No-Go 2차 판단 기준 (Phase 1 완료 시)

5개 기준 **전부** 통과해야 다음 Phase 진입.
기능 기준(1~4)이 선행. 성공률(5)은 알고리즘 품질 지표.

| # | 기준 | 통과 조건 | 실패 시 조치 |
| - | ---- | --------- | ----------- |
| 1 | result.json 스키마 유효 | VImportApp 파싱 단위 테스트 예외 없음 + 필수 필드 전부 존재 | 스키마 수정 후 재확인 |
| 2 | 경로 충돌 없음 | path 모든 복셀 `is_blocked()=False` (샘플링 금지, 전수 확인) | VoxelGrid/장애물 마킹 버그 조사 |
| 3 | 단일 경로 탐색 ≤ 30초 | vaccum_test_1 임의 1개 엔티티 기준 | 30초 초과 시 알고리즘 재검토 (heuristic, state pruning) |
| 4 | 배기 의존성 동작 | 진공 result.json 선행 생성 → 배기 preprocessing 정상 수행 | exhaust preprocessing 파이프라인 재확인 |
| 5 | 대표 시나리오 성공률 | vaccum_test_1 ≥ 80%, exhaust_test_1 ≥ 70% | 미달 시 파라미터 튜닝 후 Phase 1 재수행 (Phase 2 진입 불가) |

> **성공률 기준의 의미:** 1~4를 통과해도 성공률이 낮으면 `accuracy`, `bending_weight`, 대각 이동 구현 등 가설 파라미터 문제. Phase 2 전에 이유를 특정해야 한다.

## 7. 의사결정 포인트 3개

- **Phase 1 종료 직후** — Go/No-Go 2차 판단. 실패 시 재개발 방식 자체 재검토.
- **Phase 2 종료 직후** — API 계약 최종 확정. 이후 변경은 Client + Server 동시 수정 비용 발생.
- **Phase 3 종료 직후** — 첫 Revit import 시도. Revit 2023 환경 없으면 파싱 단위 테스트로 대체하되, 실제 임포트는 Phase 4 내 반드시 시도.

## 8. 튜닝 포인트 (Phase 4)

- `bending_optimization_weight` 값 조정 (5 고정이 최적인지 검증)
- `min_straight_distance=150mm` 실제 동작 검증
- 다중 배관 순차 라우팅 시 기존 경로 장애물 반영 반경 조정
- `accuracy` 파라미터 (진공:50, 배기:200) 반영 방식 결정
- Hookup 검증: 경사 판정 임계값 튜닝 (1% 기준이 현장 적합한지)
