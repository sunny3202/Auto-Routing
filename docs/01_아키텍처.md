## 2. 데이터 흐름 (단계별)

```
1. BIM 데이터 추출 (RevitDataExporter — SDS Tools 탭)
   Revit 모델 → D:/revit to api/Result/{title}-revit-export-{ts}.json

2. POC 데이터 준비
   equip_poc.txt / pump_poc.txt / Connectors.csv / bim_info.json

3. 사용자: Client 업로드
   fromTo 파일 (POC 데이터) + obstacle 파일 (BIM JSON) + metadata

4. arithmetic-server 수신
   → requestId 반환 → BackgroundTask로 파이프라인 실행

5. RuleSet 파이프라인 실행
   input_validation → preprocessing → routing_core(A*) → postprocessing
   각 단계: progress.json 갱신, report.json 저장

6. result.json 생성
   path 좌표 배열 (mm 단위) + attr 정보

7. Client 결과 표시
   result_report.json 렌더링, result.json 다운로드

8. SARAI Addin Import
   VImportApp: result.json → Pipe.Create()
   RImportApp: 배기 배관 + 어셈블리 조립
   → Revit 모델 업데이트 완료
```

## 3. 컴포넌트별 책임 표

| 컴포넌트 | 언어/프레임워크 | 핵심 책임 | 상태 |
| --------- | --------------- | --------- | ---- |
| AutoRouting Client | TypeScript, Next.js, Electron | 파일 업로드, 진행률 표시, 결과 리포트 | 🔧 하드코딩 제거 필요 |
| arithmetic-server | Python, FastAPI | API 서버, 파이프라인 오케스트레이션 | 🔴 신규 개발 |
| routing_core | Python, numpy | VoxelGrid, A* 탐색, 경로 단순화 | 🔴 신규 개발 |
| RuleSet 4세트 | Python | 입력검증, 전처리, 파라미터 정의, 후처리 | ✅ 재사용 |
| SARAI Addin | C#, .NET 4.8, Revit API | 앱 실행, 진공/배기 배관 Revit 임포트 | ✅ 재사용 |
| RevitDataExporter | C#, .NET 4.8, Revit API | BIM 데이터 JSON 추출 (SDS Tools) | ✅ 완성 |
