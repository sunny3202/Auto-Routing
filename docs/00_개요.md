# SARAI — 반도체 FAB Hookup 배관 설계 자동화 시스템

**작성 기준일:** 2026-04-13
**버전:** Option C (Hookup 설계사 개념 통합)

---

## 1. 시스템 정의

SARAI(Semi-conductor Auto Routing AI)는 반도체 FAB의 **진공 배관(Vacuum)**과 **배기 배관(Exhaust)**
라우팅을 자동화하는 BIM 통합 솔루션이다.

### 기존 Auto Routing과의 차이

| 구분 | 기존 개념 | SARAI 개념 |
| ---- | --------- | ---------- |
| 목표 | 점 A → 점 B 경로 탐색 | Hookup 배관 설계 자동화 |
| 결과물 | path 좌표 배열 | 경로 + 설계 품질 지표 + 검증 결과 |
| 검증 | 충돌 없음만 | 경사·dead volume·min_straight·turn 수 포함 |
| 최종 용도 | 경로 시각화 | Revit 모델 직접 임포트 (Pipe.Create) |

### 핵심 개념: Hookup 배관 설계사

SARAI는 단순 경로 탐색기가 아니라 **설계 규칙을 아는 배관 설계사**다.

- 경로 생성 → 설계 규칙 검증 → 시공 가능한 설계 제안 산출
- 진공 배관: 꺾임 최소화(dead volume), 최소 직선 구간(용접 공간), 경사 연속성
- 배기 배관: 경사 최소 1%(중력 배수), conductance 보존, 진공 배관 경로 의존

### 프로젝트 배경

- 레거시 `arithmetic.exe` + `SmartRoutingAI` 라이브러리: **바이너리 없음, 복구 불가**
- 방향: **재개발 프로젝트** (복구 아님)
- 기준: 기존 RuleSet 4세트를 명세로 삼아 동등 기능의 신규 엔진 개발
- 검증: 테스트 자산(vaccum_test_1, exhaust_test_1)을 골든 샘플로 사용

## 2. 범위 (고정)

**진공 배관 + 배기 배관 — 총 4 시나리오만 해당.**
Gas / Chemical / UPW 등 다른 utility는 이 시스템 범위 밖이다.

| 시나리오 | 대상 배관 | 골든 샘플 | 목표 성공률 |
| -------- | --------- | --------- | ----------- |
| 1단계 진공 | 장비 → 펌프 진공 배관 | vaccum_test_1 (42개) | ≥ 80% |
| 1단계 배기 | 장비 → 배기헤더 | exhaust_test_1 (44개) | ≥ 70% |
| 2단계 진공 | 2구역 진공 배관 | vaccum_test_2 (44개) | 경로 생성 여부 |
| 2단계 배기 | 2구역 배기 배관 | exhaust_test_2 | ≥ 70% |

**배기 의존성:** 1단계/2단계 배기 preprocessing은 해당 단계 진공 result.json 필수.

## 3. 보유 자산 vs 재개발 대상

### 재사용 가능 (그대로 또는 거의 그대로)

| 자산 | 재사용 | 비고 |
| ---- | ------ | ---- |
| RuleSet 4세트 (Python) | ✅ 그대로 | 입력검증, 전처리, 오프셋, 정렬, 후처리 |
| SARAI Revit Import Addin (C#) | ✅ 거의 | VImportApp, RImportApp, FamilyInstanceImporter |
| 클라이언트 UI renderer (Next.js) | 🔧 수정 | 하드코딩 제거, 실제 API 연결 |
| 테스트 자산 4세트 | ✅ 검증용 | progress/report/result JSON 골든 샘플 |
| BIM 데이터 + POC 데이터 | ✅ 그대로 | 실제 FAB 라우팅 입력 데이터 |

### 새로 만들어야 하는 것

| 대상 | 난이도 | 설명 |
| ---- | ------ | ---- |
| `routing_core` (Python 패키지) | 🔴 높음 | VoxelGrid + A* PathFinder. SmartRoutingAI 대체 |
| `arithmetic-server` (FastAPI) | 🟡 중간 | 파이프라인 오케스트레이션 + REST API. arithmetic.exe 대체 |
| Electron 메인 프로세스 | 🟡 중간 | main.js 신규 작성 (preload.d.ts 타입 맞춤) |
| Hookup 검증 레이어 | 🟡 중간 | postprocessing 확장 + 설계 품질 지표 산출 |
